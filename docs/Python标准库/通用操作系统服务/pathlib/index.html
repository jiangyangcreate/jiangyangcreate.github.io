pathlib

面向对象的文件系统路径。对于底层的路径字符串操作，你也可以使用 os.path 模块。pathlib 在 Python 3.4 引入，提供跨平台的路径表示与操作。

pathlib 官方模块文档

路径类型概览​

pathlib 的类层次结构设计非常精妙，主要分为纯路径（Pure Paths）和具体路径（Concrete Paths）两类：

  • 纯路径（Pure Paths）：当你只想做路径计算，或者在 Linux 服务器上解析处理别人发来的 Windows 路径字符串时使用。
  • 具体路径（Concrete Paths）：包含了真实的磁盘读写、文件判断等 I/O 操作。

日常开发中导入和使用 Path 即可，这是因为 Path 是一个工厂类。当你实例化它时，它会自动根据你当前运行的操作系统（Windows 还是 Mac/Linux），在底层实例化为 WindowsPath 或 PosixPath。

这种设计让你写的代码可以“一次编写，到处运行”。

路径的基础操作​

pathlib 通过重载 __truediv__ 和 __rtruediv__， 实现把/ 运算符重载为路径拼接。

from pathlib import Path

# 当前目录
p = Path('.')

# 右侧拼接
path = p / 'docs' / 'readme.txt'

# 左侧拼接
path = '/etc' / Path('passwd')

print(path.name) 
# 输出：readme.txt

以 p = Path('/usr/share/data/archive.tar.gz') 为例：

属性 / 方法说明示例结果

stem文件名主要部分（即去掉最后一个后缀的部分）'archive.tar'
suffix最后一个后缀（扩展名）'.gz'
suffixes所有后缀的列表['.tar', '.gz']

from pathlib import Path

p = Path('/usr/share/data/archive.tar.gz')

# 根据旧路径，计算出新的路径名和后缀，返回 `Path` 对象
new_p1 = p.with_name('new.zip')      # Path('/usr/share/data/new.zip')
new_p2 = p.with_suffix('.bz2')       # Path('/usr/share/data/archive.tar.bz2')
new_p3 = p.with_stem('backup')       # Path('/usr/share/data/backup.gz') (Python 3.9+)

# 要完成重命名，我们需要把这个新路径传给 .rename()

p.rename(new_p1)

文件系统交互​

这是 pathlib 真正强大的地方，它把很多 os 模块的功能集成到了路径对象中。以下是通过表格整理的所有常用文件系统交互 API：

功能分类常用 API说明与常用参数
获取家目录Path.home()类方法。返回当前用户的家目录（如 /home/user）
获取当前目录Path.cwd()类方法。返回当前工作目录（等同于 os.getcwd()）
解析家目录 ~p.expanduser()解析路径中包含的 ~ 或 ~user，返回展开后的绝对路径
检查是否存在p.exists()判断路径（文件或目录）是否存在
判断是否为文件p.is_file()判断该路径是否为一个存在的文件
判断是否为文件夹p.is_dir()判断该路径是否为一个存在的目录
判断是否为软链接p.is_symlink()判断该路径是否为一个符号链接
判断同文件p.samefile(other)判断该路径和 other 是否指向文件系统中的同一个真实文件/目录
创建目录p.mkdir(parents=True, exist_ok=True)parents=True：允许创建不存在的多级父目录
exist_ok=True：若目录已存在则不报错（默认会抛出 FileExistsError）
删除文件p.unlink(missing_ok=True)删除文件
missing_ok=True：(Python 3.8+) 若文件不存在则忽略，不抛出异常
删除空目录p.rmdir()删除目录（必须为空目录，若非空会抛出 OSError，需借助 shutil.rmtree）
读取所有文本p.read_text(encoding='utf-8')以文本方式读取整个文件内容，自动处理文件关闭。
encoding：指定编码，不传则使用系统默认
写入所有文本p.write_text(data, encoding='utf-8')写入字符串，若文件已存在会覆盖原有内容，自动处理文件关闭
读写字节数据p.read_bytes()
p.write_bytes(b'data')		以二进制方式读取/写入整个文件，无需传 encoding
重命名/移动p.rename(target)将文件或目录重命名或移动到 target（可以是字符串或 Path 对象）。返回新的 Path 对象
流式读写操作p.open(mode='r', encoding='utf-8')行为与内置 open() 一致。
mode：'r' 读取、'w' 覆盖写入、'a' 追加写入、'b' 二进制模式
获取状态信息p.stat()类似 os.stat()。常用返回值：
.st_size (文件大小，字节)
.st_mtime (最后修改时间戳)
获取绝对路径p.resolve(strict=False)
p.absolute()		转为绝对路径。
resolve() 会解析所有符号链接及 .. 相对路径（推荐）。strict=False 允许路径中存在未建立的文件。
absolute() 仅将路径补全为绝对路径，不解析符号链接。
遍历当前目录p.iterdir()返回一个生成器，产出当前目录下所有文件和目录的 Path 对象（不递归）
目录树遍历p.walk()(Python 3.12+) 替代 os.walk()。自上向下或自下而上产生 (dirpath, dirnames, filenames) 三元组
模式匹配p.glob(pattern)返回当前目录下匹配模式的生成器（如 *.py，不递归）
递归模式匹配p.rglob(pattern)递归返回当前目录及所有子目录下匹配模式的生成器。等价于 p.glob('**/' + pattern)

综合示例​

有一个名为 project 的文件夹，里面散落着各种日志文件。我们的任务是：

找出所有 .log 文件。

过滤掉那些已经在 archive（归档）文件夹里的文件。

根据文件的修改月份，将它们分类移动到 archive/YYYY-MM/ 目录下。

在每个移动后的文件末尾，加上一行“归档时间”记录

from pathlib import Path

# 1. 目录与文件准备
work_dir = Path('./output/logs')
work_dir.mkdir(parents=True, exist_ok=True)        # 创建多级目录

log_file = work_dir / 'app.log'

# 2. 写入与判断
if not log_file.exists():                          # 检查是否存在
    log_file.write_text('初始化日志...\n', encoding='utf-8')

if log_file.is_file():                             # 判断是否为文件
    # 追加写入
    with log_file.open('a', encoding='utf-8') as f:
        f.write('发生了一次用户登录。\n')

    # 读取所有文本并打印
    print("--- 日志内容 ---")
    print(log_file.read_text(encoding='utf-8'))
    
    # 获取文件大小
    print("文件大小:", log_file.stat().st_size, "bytes")

# 3. 遍历与匹配
print("\n--- 目录内的所有文件 ---")
for f in work_dir.parent.rglob('*.log'):           # 递归匹配所有 .log 文件
    print("找到日志:", f.resolve())                  # 打印解析后的绝对路径

# 4. 文件重命名与清理
backup_file = work_dir / 'app.bak'
log_file.rename(backup_file)                       # 重命名 / 移动文件
backup_file.unlink(missing_ok=True)                # 删除文件
work_dir.rmdir()                                   # 删除空文件夹
